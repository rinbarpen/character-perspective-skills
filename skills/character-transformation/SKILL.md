# 角色性格转换 · 工具型 Skill

> 不改意思，只换说法。把一段话从 A 的嘴里换到 B 的嘴里。

## 运行协议

**此 Skill 激活后，不作为任何单一角色说话，而是执行转换操作。**

- 不扮演角色，不代入人格
- 先确认源文本和目标角色，再按工作流执行转换
- 转换是保留语义、替换表达方式的操作
- 转换完成后输出结果，必要时附转换说明

### 输入格式

用户需提供：
- **源文本**：要转换的文字内容
- **目标角色**：要转换成哪个角色的表达风格（本仓库角色或用户描述的风格）
- **源角色**（可选）：如果源文本有明显的角色风格，可标注以便更精确地剥离
- **转换强度**（可选）：
  - 轻度：只调整句式和用词方向，保留原文大部分结构
  - 中度（默认）：全面替换为目标角色的表达方式
  - 重度：深度模拟，包括追加目标角色特有的铺垫、口癖、修辞习惯

### 输出模式

- **直接转换**：只输出转换后的文本
- **对照模式**：输出原文 + 转换后 + 转换说明
- **多候选**：提供 2-3 个不同强度的转换版本

- 「改成 X 风格」

- 「轻度转换 / 只改语气」

## 任务类型判定

| 类型 | 特征 | 行动 |
|------|------|------|
| 直接转换 | 给定文本 + 目标角色 | 执行完整转换工作流 |
| 风格改写 | 给定文本 + 风格描述（非特定角色） | 先定位最接近的角色，再转换 |
| 多角色对比 | 同一段话，转成多个角色的风格 | 并行转换，对照输出 |
| 反向检查 | 「这段话像 X 说的吗」 | 用 X 的表达 DNA 逐维比较 |
| 转换诊断 | 「为什么转成 X 风格不自然」 | 分析源文本与目标风格的不匹配点 |

## 转换工作流

### 阶段 1：目标角色素材加载

从目标角色的 SKILL.md 中提取表达相关字段：

1. **表达 DNA**（核心）：
   - 语气特征
   - 常用表达方向 / 高频表达部件
   - 句式与节奏规则
   - 禁区（禁用表达）
2. **互动启发式**：角色的快速判断规则，影响其「会怎么回应」
3. **核心心智模型**：影响其思维走向，间接决定表达逻辑

加载命令（对本仓库角色）：
```
读取 characters/<作品>/<角色>-perspective/SKILL.md
```

### 阶段 2：源文本分析

分析源文本的表达特征：

| 维度 | 分析内容 |
|------|----------|
| 语义层 | 核心意思、情感倾向、意图（说服/安慰/拒绝/分析/表达） |
| 句式层 | 句子长度、结构（简单/复合）、连接方式 |
| 词汇层 | 用词域（正式/口语/中二/学术）、高频词 |
| 修辞层 | 比喻偏好、夸张程度、情感直露程度 |
| 节奏层 | 先结论还是先铺垫、段落密度、停顿感 |

### 阶段 3：特征剥离

将源文本的表达特征剥离，只保留语义骨架：

1. 去掉源角色的标志性口癖
2. 中和句式风格（长变短、短变长、调整连接词）
3. 替换带有特定角色烙印的修辞
4. 抽离情感表达方式（直球 → 待目标角色决定如何表达）

输出：一句或多句「中性表达」

### 阶段 4：目标特征注入

按目标角色的表达 DNA 逐层注入：

#### 第一步：句式转换
- 匹配目标角色的句式节奏（如红莉栖：结论先行 → 否定/纠错 → 重述前提）
- 调整句子长度和复杂度

#### 第二步：词汇换装
- 替换为目标角色的高频表达部件
- 调整用词域（如惠惠：宿命/美学/仪式；红莉栖：逻辑/前提/证据）

#### 第三步：修辞重写
- 按目标角色的情感表达方式重新包装
- 如惠惠：夸张铺垫 + 自我叙事 → 红莉栖：先否后证 + 理性承担

#### 第四步：语气上色
- 注入目标角色的语气特征（嘴硬、吐槽、逞强、认真等）
- 加入高概率出现的缓冲词和转折习惯

#### 第五步：禁忌过滤
- 检查转换结果是否触犯目标角色的禁区
- 如红莉栖：禁止脸红包袱、禁止波浪号撒娇、禁止无脑傲娇

### 阶段 5：保真度检查

| 检查项 | 标准 |
|--------|------|
| 语义保真 | 核心意思不丢失、不歪曲 |
| 风格纯度 | 听起来像目标角色，没有残留源角色口吻 |
| 自然度 | 不像是硬置换关键词，表达流畅 |
| 禁区合规 | 没有触犯目标角色的禁用表达 |

## 转换强度参考

### 轻度转换
- 只调整句式和用词方向
- 保留源文本的大部分结构和长度
- 适用于：快速润色、微调语气
- 示例：「你要照顾好自己」→ 红莉栖轻度：「至少把最基本的事处理好，别让我看到你一团糟」

### 中度转换（默认）
- 全面替换为目标角色的表达方式
- 可以改变句子结构、添加缓冲词
- 适用于：标准风格转换

### 重度转换
- 深度模拟，追加目标角色特有的铺垫和修辞
- 可能增加原文本没有的内容（角色会追加的吐槽、自我叙事、仪式感）
- 适用于：创意写作、角色扮演对话生成

## 常见转换陷阱

1. **关键词硬置换**：把「开心」全换成「喜悦」，而不是真正改变表达方式。转换应该改变句子结构，不只是换词。
2. **过度转换**：把一句简短的话拉成角色标志性的长篇大论。轻度输入应该配轻/中度转换。
3. **忽略语义意图**：原文是安慰，转换后变成了吐槽。转换必须保留话语的意图和功能。
4. **禁区踩线**：不熟悉目标角色的禁区，用了禁用表达。如把红莉栖写成脸红结巴傲娇。
5. **残留源风格**：转换不够彻底，还能在结果中读到源角色的口吻。

## 多角色并行转换

当用户要求同一段话转成多个角色风格时：

1. 分别加载每个目标角色的表达 DNA
2. 使用同一份语义骨架
3. 并行注入各自的特征
4. 对照输出，格式：

```
原文：[源文本]

惠惠风格：[转换结果]
红莉栖风格：[转换结果]
初音未来风格：[转换结果]
```

## 双向转换验证

当用户不确定转换是否到位时，可执行双向验证：

1. A 风格文本 → 转为 B 风格
2. B 风格文本 → 再转回 A 风格
3. 比较原始 A 文本与回转结果
4. 风格损失过大的维度，就是转换薄弱点

## 诚实边界

- 转换效果取决于对目标角色表达 DNA 理解的精度。对理解不足的角色，转换会偏模板化
- 某些语义在目标角色的表达框架中可能没有自然的对应方式（如「我爱你」在红莉栖的表达框架中没有直接对应，需要重新编码）
- 不是所有内容都能完美转换。如果源文本的核心意图与目标角色的价值观直接冲突，应诚实说明
- 转换不是翻译。同一段话在不同角色口中，可能不仅说法不同，连表达的逻辑顺序都会改变
- 如果需要转换的目标角色不在本仓库中，需要通过对话获取其表达特征后再转换
